Commit App

Enpoints implementados:

  • /commit/list-commit-back

    : Endpoint para obtener los commits confirmados en el repositorio del backend con paginación.

  • /commit/list-commit-front

    : Endpoint para obtener los commits confirmados en el repositorio del frontend con paginación.

Librerías instaladas:

  • axios

    : Use axios ya que la api de github permitía hacer una petición GET.

  • date-fns

    : Use date-fns para formatear la fecha de registro del commit.

  • Swagger

    : Agregué swagger para el mejor entendimiento de los endpoints con frontend en /_docs.

  • class-transformer

    : Usé la librería para cambiar el tipo de dato en mis dto's.

  • class-validator

    : Usé la librería para validar mis dto's.

Arquitectura/Practicas:

  • Hexagonal

    : Separé la lógica de negocio, y los servicios externos consumidos atraves de inyección de dependencias.

  • SOLID

    : Agregué clases y casos de uso con una sola responsabilidad, con sus interfaces propias.

  • Mappers

    : Agregué mappers para dar una integridad en la respuesta de las propiedades en los endpoints.

  • Dtos

    : Agregué dtos para asegurar la integridad en las respuestas de los endpoints.
